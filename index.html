Month 2

Master LLM App Development

Build real AI-powered applications using OpenAI and Anthropic APIs. Write prompts that work reliably, get structured data out of models, make them call your functions.

SKILL 01 / 08
Prompting Fundamentals
Prompting is the craft of writing instructions that produce consistent, reliable outputs from probabilistic models. Work through all resources in order — each reinforces the others.
What to focus on
System vs user messages · specificity · chain-of-thought (think step by step) · few-shot examples · XML structuring · how small wording changes shift output quality dramatically
Resources
Anthropic Interactive Prompt Engineering Tutorial — GitHub (9 chapters with exercises) Anthropic Prompt Engineering Docs — Official Reference OpenAI Prompt Engineering Guide — Official PromptingGuide.ai — Basic to Advanced, Tool Calling, Agentic Systems
Practice: Write 5 different prompts for the same task (e.g. summarize a doc) and compare outputs side by side.
SKILL 02 / 08
Structured Outputs / JSON Schemas
In real apps you almost never want raw text — you want structured data you can parse, store, and use. Instructor is what most production AI engineers actually use.
What to focus on
Defining Pydantic models · passing schemas to the API · structured outputs vs JSON mode · handling refusals gracefully · automatic retries on validation failure
Resources
OpenAI Structured Outputs Guide — Ensures models always adhere to your JSON Schema Instructor Library — Works with OpenAI, Anthropic, Google + 15 others, auto-retries OpenAI Cookbook: Structured Outputs Introduction — Chain-of-thought, data extraction, UI gen
Practice project: Invoice/receipt parser — raw text in, structured Python object with invoice_number, amount, items, due_date out.
SKILL 03 / 08
Function / Tool Calling
Tool calling transforms an LLM from a text generator into something that can take actions. The model doesn't execute functions — it returns a structured call; your code executes it.
What to focus on
Describing functions in JSON Schema · parsing tool call responses · executing the function & feeding results back · handling cases where no tool call is needed · tool_choice: "auto"
Resources
OpenAI Function Calling Guide — 5-step flow, parallel calls, best practices Anthropic Tool Use Docs — Claude equivalent, same concepts slightly different syntax OpenAI Cookbook: How to Call Functions with Chat Models — Full runnable notebook
Practice: Assistant with 3 tools: get_weather(city), calculate(expression), search_notes(query). Wire them up, watch model decide which to call.
SKILL 04 / 08
Streaming Responses
Streaming shows output word-by-word as it's generated. Makes apps feel dramatically faster. Almost always the right choice for user-facing apps. Nobody wants to stare at a loading spinner for 10 seconds.
What to focus on
Setting stream=True · iterating over delta chunks · assembling full response from parts · wiring streaming into FastAPI endpoint using StreamingResponse
Resources
OpenAI Streaming Docs — Official API Reference Anthropic Streaming Docs — Python examples How Streaming LLM APIs Work — Simon Willison (Server-Sent Events under the hood)
SKILL 05 / 08
Conversation State
LLMs are stateless — no memory between calls. Conversation history is something you manage by sending the full message list with every request. This is fundamental.
What to focus on
The messages array structure · why you append both user and assistant messages · context window limits · basic truncation strategies (drop oldest messages, summarize history)
Resources
OpenAI Chat Completions Guide: Managing Conversations — Canonical Explanation Anthropic Messages API Docs — Equivalent reference
Practice: Multi-turn terminal chatbot. Add /reset command to clear history. Print current token count after each exchange.
SKILL 06 / 08
Cost, Latency & Token Basics
Shipping AI apps without understanding tokens is how you end up with surprise bills and slow apps. Boring but critical. Don't use GPT-4/Opus for everything.
What to focus on
What a token is (~4 chars / ¾ of a word) · input vs output tokens priced differently · context window trade-offs · latency trade-offs between smaller faster vs larger smarter models
Resources
OpenAI Pricing Page — Bookmark this, check whenever you pick a model Anthropic Pricing Page — Claude models OpenAI Tokenizer Tool — Paste text, see exact token count (use constantly while learning) Tiktoken — Python library for counting tokens in code before sending requests
SKILL 07 / 08
Failure Handling
LLM APIs fail. Rate limits get hit, responses time out, the model returns malformed JSON. Handling failures gracefully separates a demo from a production app.
What to focus on
Rate limit errors (429) & exponential backoff · timeout handling with httpx/requests · validating model output before using it · fallback strategies · never crashing on unexpected output
Resources
OpenAI Error Codes Reference — Every error type and what to do about it Anthropic Error Handling Docs — Same for Claude Tenacity — Retry Logic with Exponential Backoff (one decorator handles it all)
SKILL 08 / 08
Prompt Injection Awareness
The #1 security risk in LLM applications. Untrusted user input combined with system instructions can cause unintended actions. Know this before you ship anything.
What to focus on
Direct vs indirect injection · why system prompts aren't truly "secure" · principle of least privilege for tool access · never trust unvalidated LLM output for consequential decisions
Resources
OWASP Top 10 for LLM Apps — LLM01: Prompt Injection (authoritative, covers direct + indirect) OWASP Prompt Injection Prevention Cheat Sheet — Input validation, privilege control, output validation Evidently AI: What is Prompt Injection — Developer-focused, attack types and design patterns

Month 2 Milestone — By end of month you should be able to:

  • Write prompts that produce consistent, reliable outputs for a given task
  • Get structured JSON from any model using Pydantic + Instructor
  • Wire up tool calling so a model can call your Python functions
  • Stream responses in real time through a FastAPI endpoint
  • Manage multi-turn conversation history properly
  • Estimate the token cost of a request before sending it
  • Handle API errors, timeouts, and bad outputs without crashing
  • Explain what prompt injection is and apply basic defenses
03
Month 3

Learn RAG Properly

Build systems that let LLMs answer questions from your documents. Ingest documents, embed & store them, retrieve the right chunks, produce grounded cited answers.

SKILL 01 / 09
Embeddings
A text embedding projects text into a high-dimensional vector space. Semantically similar text ends up close together — which makes similarity search possible. The foundation everything else is built on.
What to focus on
What a vector is conceptually · why similar text produces similar vectors · cosine similarity · difference between embedding models (OpenAI vs HuggingFace sentence-transformers) · what embedding dimension means
Resources
An Intuitive Introduction to Text Embeddings — Stack Overflow Blog (best beginner explanation) Google ML Crash Course: Embeddings — Why dense vectors beat one-hot encoding Getting Started With Embeddings — HuggingFace (sentence-transformers, semantic search over FAQ) OpenAI Embeddings Guide — text-embedding-3-small and text-embedding-3-large in code
Practice: Embed 20 sentences, write nearest-neighbor search returning top 3 most similar to a query. This is the heart of RAG in miniature.
SKILL 02 / 09
Chunking
How you chunk directly affects retrieval quality. Even a perfect retrieval system fails if it searches over poorly prepared data. Chunk size ~250 tokens, 10-20% overlap is a good starting point.
What to focus on
Fixed-size with overlap as baseline · recursive chunking for structured docs · semantic chunking for better boundary detection · trade-off: too large = low precision, too small = lost context
Resources
Chunking Strategies for RAG — Weaviate (most practical guide, fixed/recursive/semantic) Chunking for RAG Best Practices — Unstructured (sizes, overlap, embedding context window limits) LangChain Text Splitters Docs — RecursiveCharacterTextSplitter, MarkdownTextSplitter, semantic splitters
Start: RecursiveCharacterTextSplitter with chunk_size=500, chunk_overlap=50. Most sensible default for most documents.
SKILL 03 / 09
Vector Databases
Choose based on situation: Chroma (local), Pinecone (managed scale), Weaviate (hybrid search), Qdrant (complex filters, self-hosted), pgvector (already on PostgreSQL).
What to focus on
Creating a collection · inserting embeddings with metadata · querying by similarity with top_k · filtering by metadata at query time. No need to understand HNSW/IVF internals.
Resources
Chroma Official Docs — In-memory/local, zero infrastructure, perfect for prototyping Pinecone Learning Center — Excellent provider-agnostic RAG tutorials Qdrant Documentation — Best open-source for production with advanced filtering pgvector — Vector search directly in PostgreSQL, no new infrastructure
Practice: Index 50-100 pages of public docs into Chroma with metadata (source URL, section title). Query function returning top 5 chunks for any question.
SKILL 04 / 09
Metadata Filtering
Raw similarity search alone isn't enough. Filtering constrains retrieval to a relevant subset. This separates toy demos from production systems where users can filter by date, source, or category.
What to focus on
Tag every chunk with metadata at ingestion (source filename, page number, section, date, category) · use those fields to filter results at query time
Resources
Pinecone: Metadata Filtering Guide — Clear explanation with code examples LlamaIndex: Metadata Filters Guide — Applying filters at query time in LlamaIndex pipelines
SKILL 05 / 09
Reranking
Two-stage: embed and search (fast, approximate) → rerank top-k (slower, more accurate). Dramatically better retrieval quality with only a modest latency cost.
What to focus on
Two-stage retrieve-then-rerank pattern · bi-encoder (first-stage embedding search) vs cross-encoder (reranking) · latency/quality trade-off of reranking top-20 vs top-5
Resources
Cohere Reranking Docs — Full workflow including structured data like emails & JSON. Single line to add. LangChain: Cohere Reranker Integration — ContextualCompressionRetriever
SKILL 06 / 09
Retrieval Quality Issues
Most RAG failures aren't model failures — they're retrieval failures. Understanding how retrieval goes wrong is essential for debugging real systems.
Common issues + fixes
Semantic drift → query rewriting / HyDE | Chunk boundaries → increase overlap / semantic chunking | Missing metadata context → use metadata filtering | Top-k too small → increase top_k at retrieval, reduce after reranking
Resources
LangChain: Query Transformations — Rewriting, step-back prompting, HyDE Pinecone: Improving Retrieval Quality — Common failure modes with fixes
SKILL 07 / 09
Hallucination Reduction
RAG dramatically reduces hallucinations vs vanilla LLMs but doesn't eliminate them. Retrieval failures, bad chunks, and conflicting info can still cause the model to make things up.
What to focus on
Prompt grounding (answer only from provided context) · "I don't know" when answer isn't there · confidence threshold before surfacing responses · validate retrieval quality before blaming the LLM
Resources
Reducing LLM Hallucinations: A Developer's Guide — Zep (prompt grounding, CoT, output verification) 5 Ways to Reduce LLM Hallucinations — Voiceflow (RAG + CoT + guardrails combined)
SKILL 08 / 09
Citations & Grounding
A grounded RAG system doesn't just answer — it tells you where the answer came from. Critical for user trust and for debugging.
What to focus on
Pass chunk metadata (source filename, page number, URL) into your prompt · instruct model to reference sources in its answer · surface sources in your UI/API response
Resources
Anthropic: Giving Claude Sources — How to prompt Claude for cited responses with source references LangChain: RAG with Sources — Return source documents alongside answers in a LangChain pipeline
SKILL 09 / 09
RAG Frameworks: LlamaIndex & LangChain
LlamaIndex: optimized for search & indexing, working prototype in an afternoon. LangChain: shines for orchestration, multi-agent, tool calling. Start with LlamaIndex for RAG in Month 3, LangChain in Month 4.
LlamaIndex 5 key stages
Loading · Indexing · Storing · Querying · Evaluating
Resources
LlamaIndex: Introduction to RAG — 5 key stages and how LlamaIndex handles each LlamaIndex Starter Tutorial — Working RAG system in under 30 lines (official quickstart) LangChain: Build a RAG Agent — From 40-line minimal version to full pipeline with reranking
Practice: "Chat with your docs" — ingest 10-20 PDFs, FastAPI endpoint, top 5 chunks with reranking, cited answer from Claude/OpenAI. This is a real portfolio piece.

Month 3 Milestone — By end of month you should be able to:

  • Explain what an embedding is and why similar text produces similar vectors
  • Chunk any document intelligently using appropriate strategies
  • Store and query embeddings in a vector database with metadata filtering
  • Add a reranking step to improve retrieval quality
  • Debug common retrieval failures systematically
  • Build complete end-to-end RAG pipeline with grounded, cited answers

Month 4

Agents, Tools, Workflows & Evals

Build AI systems that take sequences of actions autonomously. Know when agents are the wrong choice. Measure the performance of everything you build.

SKILL 01 / 08
Agent Loops
An agent is not magic — it's a while loop with an LLM making the branching decisions. The "thinking" is in the prompt, the "branching" is when the agent chooses between tools, the "doing" is when external functions are called.
What to focus on
Perceive → Plan → Act → Observe cycle · how the agent loop terminates · what happens when a tool call fails · why agents are just while loops with LLM branching
Resources
Anthropic: Building Effective Agents — Single best piece on agents in production (read this first) OpenAI: A Practical Guide to Building Agents — Official PDF, patterns, guardrails, safety The Open Source LLM Agent Handbook — freeCodeCamp (LangGraph, CrewAI, planning, memory, tools) Intro to LangGraph — LangChain Academy (free official course, state/memory/human-in-the-loop)
Build an agent from scratch without any framework first — just the raw API. This is the most valuable thing you can do to understand what frameworks are abstracting.
SKILL 02 / 08
Tool Selection & Design
Tool descriptions are the user manual for the LLM. If the manual is vague, the LLM misuses the tool. A poorly described tool will be called wrong, at the wrong time, or ignored entirely.
What to focus on
Self-explanatory verb names · descriptions explaining WHEN to call (not just what it does) · keep params minimal and well-typed · design with the LLM as the caller in mind
Resources
OpenAI: Function Calling Best Practices — Naming conventions and parameter documentation patterns Anthropic: Tool Use Best Practices — When to let model choose vs forcing a specific tool
Test every description: "If I had no docs and only this JSON schema, would I know exactly when and how to call this?" If not, rewrite it.
SKILL 03 / 08
State Management
In LangGraph, state is a shared memory object that flows through the graph — storing messages, variables, intermediate results, and decision history. Key to multi-turn tasks and clean handoffs.
What to focus on
Defining state schemas with TypedDict · how reducers work for merging parallel updates · in-memory state vs persisted checkpointing · human-in-the-loop pauses by inspecting/modifying state mid-execution
Resources
LangGraph Official Docs: State Management — State schemas, reducers, how state flows through nodes DataCamp: LangGraph Agents Tutorial — State, nodes, edges, building up to stateful agents with persistent memory Real Python: LangGraph in Python — Complete stateful LangGraph agent with detailed state graph explanation
SKILL 04 / 08
Retries & Failure Handling in Agents
Agents fail differently to regular LLM calls. A bad tool call mid-loop can corrupt state, cause infinite loops, or silently produce wrong answers. Explicit strategies for all of these are required.
What to focus on
Maximum iteration limits to prevent infinite loops · per-tool retry with exponential backoff · catching and logging exceptions at the tool layer without crashing the agent · when to surface failure vs retry silently
Resources
LangGraph: Error Handling and Retries — Automatic error handling and retry logic at the tool node level OpenAI Practical Agents Guide: Guardrails Section — Layered defense: LLM checks + regex + moderation APIs
SKILL 05 / 08
When NOT to Use Agents
One of the most important and most overlooked skills. Agents are slow, expensive, unpredictable, and hard to debug. Knowing when to use something simpler is a sign of good judgment.
Decision framework — memorize this
Single LLM call → if task can be solved in one prompt | Workflow → if steps are fixed & predictable | Agent → ONLY if number of steps is genuinely unpredictable & requires dynamic decision-making
Resources
Anthropic: Building Effective Agents — When to use agents, the authoritative answer Simon Willison: Designing Agentic Loops — Senior engineer's take on when complexity is justified
A chain of 3 fixed LLM calls will always be faster, cheaper, and more debuggable than an agent that could make 3 calls. Reserve agents for genuinely open-ended tasks.
SKILL 06 / 08
Multi-Step Workflows
Between "single prompt" and "full agent" lies a vast productive middle ground. Workflows trade latency for higher accuracy by making each LLM call an easier, more focused task.
Common patterns
Prompt chaining (output of one = input to next) · Routing (classify → specialized handler) · Parallelization (multiple calls simultaneously, then aggregate) · Orchestrator-subagent (one LLM plans, others execute)
Resources
Anthropic: Workflow Patterns — All main patterns with diagrams and code examples LangGraph: Multi-Agent Networks — Supervisor and handoff patterns
Practice: 3-step pipeline — extract facts from article → generate tweet/LinkedIn/summary in parallel → score all three and pick the best. No agent required.
SKILL 07 / 08
Evaluation Harnesses
Evals are how you know if your system is actually working — not just on examples you tested by hand, but systematically across hundreds of inputs. Every prompt change without evals is a gamble.
What to focus on
Golden test sets of 20-50 inputs with expected outputs · deterministic scoring (string match, JSON schema validation) · LLM-as-judge for open-ended outputs · running evals automatically on every change
Resources
DeepEval — Open-source, pytest-inspired, 50+ built-in metrics (hallucination, relevancy, factual consistency) Promptfoo — CLI test suite, multi-prompt/model comparison, CI/CD integration + red teaming LangSmith — Tracing, debugging, evaluation (free tier generous, great for debugging agent loops) Ragas — RAG-specialized: faithfulness, answer relevancy, context precision and recall
SKILL 08 / 08
Task Success Metrics
Beyond automated evals, you need metrics that tell you whether your agent is actually accomplishing its goal.
What to focus on
Process metrics (did agent call the right tool?) vs outcome metrics (did the task succeed?) · define clear success criteria BEFORE building · LLM-as-judge for long-form answers that resist exact matching
Resources
Your AI Product Needs Evals — Hamel Husain (practical, production AI at scale) OpenAI Evals Framework — Large library of community-contributed eval patterns to adapt
Practice: Take your Month 3 RAG pipeline. Create 30 Q&A pairs. Score each answer with DeepEval. Change one thing (chunk size, model, top-k) and re-run to see if it improved.

Month 4 Milestone — By end of month you should be able to:

  • Explain what an agent loop is and implement one from scratch without a framework
  • Write tool descriptions that get selected correctly and reliably
  • Manage agent state properly using LangGraph or equivalent
  • Handle failures inside agent loops without crashing
  • Decide confidently whether a task needs agent, workflow, or single prompt
  • Build multi-step workflows that chain, route, and parallelize LLM calls
  • Write automated evals that catch regressions when prompts/models change
  • Define and measure task success metrics for any AI system you build

Month 5

Deployment, Product Thinking & Reliability

Take everything you've built and make it production-ready. Real users, real traffic, real failures. This is where most AI engineers stall — and what companies actually pay for.

SKILL 01 / 08
FastAPI Production Patterns
A single uvicorn process with --reload is fine for building. In production it becomes the bottleneck the moment real traffic arrives.
What to focus on
Gunicorn with Uvicorn workers (not bare Uvicorn) · health check endpoints · CORS middleware · async database sessions · background tasks for non-blocking responses
Resources
FastAPI Deployment Docs — Official (Uvicorn workers, Gunicorn, Docker) FastAPI Production Deployment Guide — Gunicorn config, Nginx reverse proxy, health checks, rate limiting FastAPI Best Practices for Production 2026 — Async DB pooling, Redis caching, JWT auth, background tasks
SKILL 02 / 08
Docker
Docker solves "it works on my machine." Consistent deployments, no dependency conflicts, easy scaling. Containerize your FastAPI + LLM app and deploy it anywhere.
What to focus on
Writing a Dockerfile for Python/FastAPI · multi-stage builds (keep images small) · Docker Compose for multi-service (app + DB + Redis) · env vars for secrets · .dockerignore to avoid leaking sensitive files
Resources
Docker Official Getting Started Guide — Images, containers, Dockerfiles, Docker Compose Build & Deploy Multi-Agent AI with Python and Docker — freeCodeCamp end-to-end tutorial Deploy LLM Applications Using Docker — DataCamp (specifically for LLM apps with RAG pipelines) Docker Containerization for LLM Apps — ApXML (base images, multi-stage builds, multi-service)
Practice: Containerize your Month 3 RAG app. docker-compose.yml: FastAPI + vector DB + Redis. docker compose up starts everything.
SKILL 03 / 08
Background Jobs & Queues
LLM calls are slow. If a user waits 30 seconds, they leave. Accept the request immediately, process async, notify when done.
What to focus on
FastAPI BackgroundTasks for quick fire-and-forget · Celery for heavier tasks · Redis as message broker · handling task failures and retries · returning job status to the user
Resources
Celery Official Getting Started Guide — Standard Python task queue, task definition, worker management FastAPI Background Tasks Docs — Built-in lightweight tasks, use for simple fire-and-forget
SKILL 04 / 08
Auth & API Key Security
Without auth, anyone can use your endpoints and burn through your LLM credits. You'll wake up to a $5,000 bill.
What to focus on
JWT tokens for user auth · API key management for service-to-service · rate limiting per user/key · never store secrets in code (use env vars) · authentication (who are you) vs authorization (what can you do)
Resources
FastAPI Security Docs — OAuth2, JWT tokens, API keys, dependency-based auth (full tutorial) OWASP API Security Top 10 — Broken authentication, injection, mass assignment Auth0: API Auth Best Practices — Practical guide to authentication and authorization in APIs
SKILL 05 / 08
Logging & Observability
LLM apps can return 200 status code and still produce a useless or hallucinated answer — traditional monitoring doesn't catch this. You need LLM-specific observability.
What to focus on
Trace every LLM call (input prompt, output, tokens, latency, cost) · structured JSON logging · dashboards showing request volume, error rates, cost per day · alerting when things break or costs spike
Resources
Langfuse — Open-source LLM observability (prompt versioning, eval, LLM-as-judge, integrates OpenAI/Anthropic/LangChain/LlamaIndex) LangSmith — Tracing, debugging, monitoring dashboards. One env var to set up with LangChain. Free tier generous. Python Structlog — Structured JSON logging, far better than print() or basic logging for production
SKILL 06 / 08
Prompt Version Management
In production, your prompts are code. They need version control, testing, and rollback ability. Changing a prompt without tracking what you changed is how you break things and can't figure out why.
What to focus on
Store prompts outside your application code · version every prompt change · A/B test prompt variants in production · have a rollback strategy when a new prompt performs worse · deploy prompt changes without redeploying the app
Resources
Langfuse Prompt Management — Centralized versioning, built-in playground for testing Anthropic Prompt Management Best Practices — Organizing, iterating, managing prompts at scale
SKILL 07 / 08
Cost Monitoring & Rate Limits
Without cost controls, a traffic spike or prompt bug can burn through hundreds of dollars in minutes.
What to focus on
Hard spending limits per day/month · per-user rate limits in your API · use cheaper models for simple tasks · cache repeated identical requests with Redis · monitor cost per request to catch expensive prompts early
Resources
OpenAI Usage Dashboard — Track spending by model/day, set usage limits Anthropic Usage Dashboard (console.anthropic.com) — Claude API usage tracking Helicone — Proxy-based observability, automatic cost tracking, one line of code (change base URL) LiteLLM — Unified interface for 100+ LLM providers, budget management, rate limiting, spend tracking
SKILL 08 / 08
Caching
If 20% of users ask similar questions, you're paying for the same LLM call 20 times. Caching is the simplest way to reduce costs and latency simultaneously.
What to focus on
Exact-match caching for identical prompts · semantic caching for similar queries · cache invalidation strategies (TTL-based is simplest) · measuring cache hit rates to understand real cost savings
Resources
Redis Official Docs — Standard in-memory data store, fast and simple for LLM response caching GPTCache — Semantic caching for LLMs using embedding similarity (not just exact match)

Month 5 Milestone — By end of month you should be able to:

  • Deploy a FastAPI + LLM app in Docker with proper production configuration
  • Handle long-running tasks with background jobs and queues
  • Secure your API with auth, rate limits, and API key management
  • Trace and debug LLM calls using Langfuse or LangSmith
  • Manage prompts with version control and rollback capability
  • Monitor costs in real time and set spending limits
  • Cache LLM responses to reduce latency and cost

What to actually take away from all this

TAKEAWAY 01

Pick one project from each month and BUILD it. Not read about it. Not watch a tutorial. Build it, break it, fix it, deploy it, put it on GitHub. The engineers who get hired show what they've built, not what they've studied.

TAKEAWAY 02

Start sharing what you learn. Write about it on X, LinkedIn, anywhere. Teaching is the fastest way to learn and it builds your reputation at the same time. The best opportunities come from people who were visible, not from people who applied to 500 job listings.

TAKEAWAY 03

Don't wait until you feel ready. You will never feel ready. The gap between "I'm learning" and "I'm building" is where most people get stuck forever. Start applying and freelancing the moment you have working projects — even if they're not perfect.

TAKEAWAY 04

The market doesn't reward perfection. It rewards people who can ship. 6 months is enough to change everything if you actually put in the work. Just never stop building and never stop learning.
